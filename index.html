JavaScript
Определение

JavaScript — это высокоуровневый, интерпретируемый язык программирования, который используется для создания интерактивных и динамичных веб-страниц. Он является одним из основных компонентов веб-разработки, наряду с HTML и CSS. JavaScript позволяет разработчикам добавлять функциональность на клиентской стороне, такую как обработка событий, манипуляция с элементами DOM, анимация и взаимодействие с сервером через AJAX.

JavaScript поддерживает объектно-ориентированное, функциональное и императивное программирование. Он также имеет богатую экосистему библиотек и фреймворков, таких как React, Angular и Vue.js, которые упрощают разработку сложных веб-приложений. С помощью Node.js JavaScript также может использоваться на серверной стороне, что позволяет создавать полностековые приложения с использованием одного языка.

История

JavaScript был создан в 1995 году Бренданом Айком, работавшим в компании Netscape. Изначально язык назывался Mocha, затем был переименован в LiveScript, а в конечном итоге получил название JavaScript, чтобы воспользоваться популярностью языка Java, хотя на самом деле они не имеют много общего.

В 1996 году JavaScript был стандартизирован в рамках ECMA International и получил название ECMAScript. Первая версия стандарта, ECMAScript 1, была выпущена в 1997 году. С тех пор язык продолжал развиваться, и были выпущены новые версии, добавляющие различные функции и улучшения.

В 2005 году JavaScript стал особенно популярным благодаря появлению AJAX, что позволило создавать более интерактивные веб-приложения. В 2009 году была выпущена версия ECMAScript 5, которая добавила множество новых возможностей, таких как JSON и улучшенная работа с массивами.

С выходом ECMAScript 6 (или ES6) в 2015 году JavaScript получил значительные улучшения, включая стрелочные функции, классы, модули и промисы, что сделало его более мощным и удобным для разработки.

Сегодня JavaScript является одним из самых популярных языков программирования в мире, используемым как на клиентской, так и на серверной стороне, благодаря таким технологиям, как Node.js. Он продолжает развиваться, и новые версии стандарта продолжают выходить, добавляя новые возможности и улучшения.

Преимущества
кросс-платформенность

JavaScript работает на всех современных браузерах и платформах, что позволяет разработчикам создавать приложения, которые могут работать на различных устройствах.

интерактивность

JavaScript позволяет добавлять интерактивные элементы на веб-страницы, такие как анимации, формы и динамическое обновление контента без перезагрузки страницы

широкая экосистема

Существует множество библиотек и фреймворков (например, React, Angular, Vue.js), которые упрощают разработку и позволяют создавать сложные приложения быстрее и эффективнее.

поддержка асинхронного программирования

JavaScript поддерживает асинхронные операции, что позволяет выполнять задачи, такие как запросы к серверу, не блокируя основной поток выполнения.

объектно-ориентированное программирование

JavaScript поддерживает объектно-ориентированные парадигмы, что позволяет организовывать код более структурированно и удобно.

серверная разработка

С помощью Node.js JavaScript можно использовать для серверной разработки, что позволяет создавать полностековые приложения с использованием одного языка.

большое сообщество

JavaScript имеет огромное сообщество разработчиков, что обеспечивает доступ к множеству ресурсов, учебных материалов и поддержки.

совместимость с другими технологиями

JavaScript легко интегрируется с другими языками и технологиями, такими как HTML и CSS, а также с различными API и базами данных.

постоянное развитие

Язык постоянно обновляется и улучшается, что позволяет разработчикам использовать современные функции и подходы в своих проектах.

Недостатки
проблемы с производительностью

JavaScript может быть медленнее по сравнению с компилируемыми языками, такими как C++ или Java, особенно при выполнении сложных вычислений или обработки больших объемов данных.

отсутствие строгой типизации

JavaScript является динамически типизированным языком, что может привести к ошибкам, связанным с типами данных, которые не обнаруживаются до выполнения программы. Это может усложнить отладку и поддержку кода.

безопасность

JavaScript может быть уязвим для различных атак, таких как XSS (межсайтовый скриптинг) и CSRF (межсайтовая подделка запроса). Разработчики должны быть внимательны к безопасности своих приложений и использовать соответствующие меры защиты.

проблемы с совместимостью

Хотя JavaScript работает на большинстве современных браузеров, различия в реализации могут привести к проблемам совместимости. Разработчики должны тестировать свои приложения на разных платформах и браузерах.

сложность в управлении большими проектами

При разработке крупных приложений код может стать трудным для понимания и поддержки. Это может потребовать использования дополнительных инструментов и архитектурных подходов, таких как модульность и фреймворки.

асинхронность

Хотя асинхронное программирование является преимуществом, оно также может усложнить код, особенно для разработчиков, не знакомых с этой концепцией. Обработка колбеков и промисов может привести к "адскому колбеку" (callback hell).

зависимость от браузера

JavaScript зависит от браузера для выполнения, что может ограничивать его использование в некоторых средах, таких как серверные приложения, хотя это и решается с помощью Node.js.

JavaScript и frontEnd
Динамическое обновление контента

JavaScript позволяет изменять содержимое веб-страниц без необходимости перезагрузки. Это достигается с помощью манипуляции с DOM (Document Object Model), что позволяет добавлять, удалять или изменять элементы на странице в реальном времени.

интерактивные элементы

С помощью JavaScript можно создавать интерактивные элементы, такие как кнопки, формы, слайдеры, модальные окна и другие компоненты, которые улучшают пользовательский опыт.

анимации и эффекты

JavaScript позволяет добавлять анимации и визуальные эффекты, такие как плавные переходы, изменение стилей и перемещение элементов, что делает интерфейс более привлекательным и отзывчивым.

обработка событий

JavaScript позволяет реагировать на действия пользователя, такие как клики, наведение мыши, ввод текста и другие события. Это позволяет создавать интерактивные приложения, которые реагируют на действия пользователя.

взаимодействие с API

JavaScript может использоваться для выполнения асинхронных запросов к серверу (например, с помощью Fetch API или XMLHttpRequest), что позволяет загружать данные без перезагрузки страницы. Это особенно полезно для создания одностраничных приложений (SPA).

фреймворки и библиотеки

JavaScript имеет множество фреймворков и библиотек, таких как React, Angular и Vue.js, которые упрощают разработку сложных пользовательских интерфейсов и помогают организовать код более структурированно.

валидация форм

JavaScript может использоваться для проверки данных, введенных пользователем в формы, до их отправки на сервер. Это позволяет улучшить пользовательский опыт и снизить количество ошибок.

создание одностраничных приложений (SPA)

JavaScript позволяет создавать приложения, которые загружают контент динамически, что делает их более быстрыми и отзывчивыми. Это достигается с помощью маршрутизации и управления состоянием приложения.

Компиляция
определение

Компиляция — это процесс преобразования исходного кода, написанного на языке программирования, в машинный код или байт-код, который может быть выполнен компьютером. В контексте JavaScript, компиляция может происходить в нескольких формах:

интерпретация

JavaScript традиционно интерпретируется, что означает, что код выполняется построчно в момент его выполнения. Это позволяет быстро разрабатывать и тестировать код, но может привести к снижению производительности по сравнению с компилируемыми языками.

JIT-компиляция (Just-In-Time)

Современные JavaScript-движки, такие как V8 (используется в Google Chrome и Node.js), используют JIT-компиляцию. Это означает, что код компилируется в машинный код во время выполнения, что позволяет достичь более высокой производительности, чем при чистой интерпретации. JIT-компиляторы анализируют код и оптимизируют его на лету, что может значительно ускорить выполнение.

транспиляция

В некоторых случаях разработчики используют транспиляторы (например, Babel), чтобы преобразовать код, написанный на более современных версиях JavaScript (ES6 и выше), в более старые версии, которые поддерживаются большим количеством браузеров. Это позволяет использовать новые функции языка, не беспокоясь о совместимости.

предварительная компиляция

Некоторые инструменты и фреймворки могут выполнять предварительную компиляцию шаблонов или компонентов, что позволяет улучшить производительность и упростить разработку.

Определение

Переменные в JavaScript — это именованные контейнеры, которые используются для хранения данных, которые могут изменяться в процессе выполнения программы. Они позволяют разработчикам сохранять значения и обращаться к ним по имени, что делает код более читаемым и управляемым.
переменные объявляются с помощь "let" и "const"
в старом коде можно встретить объявление с помощью "var"

декларирование let x;

присваивание x = 5;

обычно делается и декларирование и присваивание в одной строке
let x = 5;
let a = 1, b=2, c=3;

константы сразу декларируются с присваением

var
определение

В JavaScript var — это один из способов объявления переменных, который был доступен с самого начала языка. Однако с появлением let и const в стандарте ES6 (ECMAScript 2015) использование var стало менее распространенным. Давайте рассмотрим основные характеристики и особенности var.

область видимости:

Переменные, объявленные с помощью var, имеют функциональную или глобальную область видимости. Это означает, что если переменная объявлена внутри функции, она доступна только в этой функции. Если она объявлена вне функции, она доступна во всей программе.

var x = 10; function example() {
 var y = 20;
 console.log(x); // 10
 console.log(y); // 20
}
example();
console.log(y); // Ошибка: y не определена

переназначение:

Переменные, объявленные с помощью var, могут быть переназначены, то есть вы можете изменить их значение после инициализации.

var count = 1;
console.log(count); // 1
count = 2; // Это допустимо
console.log(count); // 2

повторное объявление:

Переменные, объявленные с помощью var, могут быть повторно объявлены в одной и той же области видимости без ошибок.

var x = 10;
var x = 20; // Это допустимо
console.log(x); // 20

поддержка временной мертвой зоны (TDZ):

В отличие от let и const, переменные, объявленные с помощью var, могут быть использованы до их объявления, но их значение будет undefined.

console.log(z); // undefined
var z = 10;

рекомендации по использованию var:

Используйте var только в тех случаях, когда вам действительно нужна функциональная область видимости.
В большинстве случаев рекомендуется использовать let и const, чтобы избежать проблем с областью видимости и потенциальных ошибок в коде.
let и const обеспечивают более предсказуемое поведение и помогают избежать ошибок, связанных с повторным объявлением переменных.

let
определение

let — это один из способов объявления переменных в JavaScript, который был введен в стандарте ES6 (ECMAScript 2015). Он предоставляет более современный и безопасный способ работы с переменными по сравнению с устаревшим var. Давайте рассмотрим основные характеристики и особенности let.

блочная область видимость

Переменные, объявленные с помощью let, имеют блочную область видимости. Это означает, что они доступны только в пределах блока кода, в котором были объявлены (например, внутри фигурных скобок {}).

let a = 5;
 if (true) {
  let b = 10;
  console.log(a); // 5
  console.log(b); // 10
 }
console.log(b); // Ошибка: b не определена

переназначение

Переменные, объявленные с помощью let, могут быть переназначены, то есть вы можете изменить их значение после инициализации.

let count = 1;
console.log(count); // 1
count = 2; // Это допустимо
console.log(count); // 2

не могут быть повторно объявлены в одной области видимости:

let x = 10;
let x = 20; // Ошибка: Identifier 'x' has already been declared

поддержка временной мертвой зоны (TDZ):

Переменные, объявленные с помощью let, не могут быть использованы до их объявления. Это означает, что если вы попытаетесь обратиться к переменной до ее объявления, вы получите ошибку.

console.log(y); // Ошибка: Cannot access 'y' before initialization
let y = 10;

рекомендации по использованию let:

Используйте let для переменных, значения которых могут изменяться в процессе выполнения программы.
Предпочитайте let вместо var, чтобы избежать проблем с областью видимости и потенциальных ошибок в коде.

const
определение

const — это еще один способ объявления переменных в JavaScript, который был введен в стандарте ES6 (ECMAScript 2015). Он используется для объявления констант, то есть переменных, значение которых не должно изменяться после инициализации. Давайте рассмотрим основные характеристики и особенности const.

блочная область видимости:

Переменные, объявленные с помощью const, имеют блочную область видимости, аналогично let. Это означает, что они доступны только в пределах блока кода, в котором были объявлены.

const a = 5;
if (true) {
 const b = 10;
 console.log(a); // 5
 console.log(b); // 10
}
console.log(b); // Ошибка: b не определена

неизменяемость ссылки:

Переменные, объявленные с помощью const, не могут быть переназначены. Это означает, что вы не можете изменить значение переменной после ее инициализации. Однако, если переменная ссылается на объект или массив, вы можете изменять свойства этого объекта или элементы массива.

const count = 1;
// count = 2; // Ошибка: Assignment to constant variable
const arr = [1, 2, 3];
arr.push(4); // Это допустимо
console.log(arr); // [1, 2, 3, 4]

не могут быть повторно объявлены в одной области видимости:

Если вы попытаетесь объявить переменную с тем же именем в одной и той же области видимости, это приведет к ошибке.

const x = 10;
const x = 20; // Ошибка: Identifier 'x' has already been declared

поддержка временной мертвой зоны (TDZ):

Переменные, объявленные с помощью const, не могут быть использованы до их объявления. Если вы попытаетесь обратиться к переменной до ее объявления, вы получите ошибку.

console.log(y); // Ошибка: Cannot access 'y' before initialization
const y = 10;

рекомендации по использованию const:

Используйте const для переменных, значение которых не должно изменяться в процессе выполнения программы.
Предпочитайте const вместо var и let, когда вы уверены, что значение переменной не будет изменяться, чтобы сделать код более предсказуемым и понятным.

ТИПЫ ДАННЫХ
Примитивные типы данных
number

Представляет числовые значения, как целые, так и дробные.

const num = 42;
const pi = 3.14;
console.log(typeof num) // number

string

Представляет строковые значения. Всё что записано в ковычках является строкой.

const firstName = "Dmitrii";
console.log(typeof firstName); // string
const age = '41';
console.log(typeof age); // string

boolean

Представляет логические значения: true или false.

const isActive = true;
console.log(typeof isActive); // boolean
const isComplete = false;
console.log(typeof isComplete); // boolean

null

Представляет намеренное отсутствие значения. Это специальное значение, которое указывает на "ничто".

const city = null; console.log(typeof city); // object историческая ошибка, оставленная для обратной совместимоси

undefined

Представляет отсутствие значения, значение не определено

let day;
console.log(day) // undefined
console.log(typeof day); // undefined
day = undefined;
console.log(day); // undefined

symbol (введен в ES6):

Представляет уникальные и неизменяемые значения, которые могут быть использованы в качестве идентификаторов для свойств объектов.

const uniqueId = Symbol('id');
console.log(typeof uniqueId); // symbol

bigInt (введен в ES11):

Представляет целые числа произвольной длины. Используется для работы с числами, которые превышают пределы типа Number.

const bigNumber = BigInt(123456789012345678901234567890);
console.log(typeof bigNumber); // bigint
const bigNumberN = 123456789012345678901234567890n;
console.log(typeof bigNumberN); // bigint

Ссылочные типы данных
object
объекты

Представляет коллекцию свойств, где каждое свойство имеет ключ и значение. Объекты могут содержать любые типы данных, включая другие объекты.

const person = {
 name: "Alice",
 age: 25,
 isStudent: false
 isActive
};
console.log(typeof person); // object

массивы

Специальный тип объекта, который представляет упорядоченные коллекции значений. Элементы массива могут быть любого типа.

let fruits = ["apple", "banana", "cherry"];
console.log(typeof fruits) // object

функции

Функции в JavaScript также являются объектами. Они могут быть присвоены переменным, переданы как аргументы и возвращены из других функций.

function greet() {
 console.log("Hello!");
}
console.log(typeof greet); // function, но относится к объетам

Рекомендации по использованию типов данных

Используйте примитивные типы для простых значений, таких как числа и строки.
Используйте объекты и массивы для хранения более сложных структур данных.
При работе с большими числами, которые выходят за пределы Number, используйте BigInt.
Для уникальных идентификаторов используйте Symbol.

МАССИВЫ
Методы массивов
добавление элементов
push(): Этот метод добавляет один или несколько элементов в конец массива и возвращает новую длину массива.

let fruits = ['apple', 'banana'];
console.log(fruits); //['apple', 'banana]
fruits.push('orange');
console.log(fruits); // ['apple', 'banana', 'orange']

unshift(): Этот метод добавляет один или несколько элементов в начало массива и также возвращает новую длину массива.

let fruits = ['apple', 'banana'];
console.log(fruits); // ['apple', 'banana']
fruits.unshift('orange');
console.log(fruits); // ['orange', 'apple', 'banana']

splice(): Этот метод может использоваться для добавления элементов в произвольное место массива. Он изменяет содержимое массива, удаляя или заменяя существующие элементы и/или добавляя новые.

let fruits = ['apple', 'banana'];
console.log(fruits); // ['apple', 'banana']
fruits.splice(1, 0, 'orange'); // добавляет 'orange' на индекс 1
console.log(fruits); // ['apple', 'orange', 'banana']

Расширение массива с помощью оператора распространения (...): Вы можете создать новый массив, добавляя элементы к существующему массиву.

let fruits = ['apple', 'banana'];
console.log(fruits); // ['apple', 'banana']
let newFruits = [...fruits, 'orange'];
console.log(newFruits); // ['apple', 'banana', 'orange']

concat(): Этот метод объединяет два или более массивов и возвращает новый массив. Вы можете использовать его для добавления элементов.

let fruits = ['apple', 'banana'];
console.log(fruits); // ['apple', 'banana']
let newFruits = fruits.concat('orange');
console.log(newFruits); // ['apple', 'banana', 'orange']

удаление элементов
pop(): Удаляет последний элемент из массива и возвращает его. Если массив пуст, возвращает undefined.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
let lastFruit = fruits.pop();
console.log(lastFruit) // 'orange'
console.log(fruits); // ['apple', 'banana']

shift(): Удаляет первый элемент из массива и возвращает его. Если массив пуст, возвращает undefined.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
let firstFruit = fruits.shift();
console.log(firstFruit); // 'apple'
console.log(fruits); // ['banana', 'orange']

splice(): Может использоваться для удаления элементов из произвольного места массива. Он изменяет содержимое массива, удаляя или заменяя существующие элементы и/или добавляя новые.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
let removedFruits = fruits.splice(1, 1); // Удаляет 1 элемент с индекса 1
console.log(fruits); // ['apple', 'orange']
console.log(removedFruits); // ['banana']

filter(): Создает новый массив с элементами, которые прошли проверку, заданную в предоставленной функции. Это позволяет "удалить" элементы, которые не соответствуют условию.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
let filteredFruits = fruits.filter(fruit => fruit !== 'banana'); // Удаляет 'banana'
console.log(filteredFruits); // ['apple', 'orange']
console.log(fruits); // ['apple', 'banana', 'orange']

slice(): Хотя этот метод не удаляет элементы, он может использоваться для создания нового массива, исключая определенные элементы.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
let newFruits = fruits.slice(1); // Создает новый массив, начиная с индекса 1
console.log(newFruits); // ['banana', 'orange']
console.log(fruits); // ['apple', 'banana', 'orange']

изменение элементов массива
Прямое присваивание: Вы можете изменить элемент массива, обратившись к его индексу и присвоив новое значение.
let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
fruits[1] = 'kiwi'; // Изменяет 'banana' на 'kiwi'
console.log(fruits); // ['apple', 'kiwi', 'orange']
splice(): Этот метод может использоваться не только для добавления и удаления элементов, но и для замены существующих элементов.

let fruits = ['apple', 'banana', 'orange', 'strawberry'];
console.log(fruits); // ['apple', 'banana', 'orange', 'strawberry']
fruits.splice(1, 2, 'kiwi', 'grape'); // Удаляет 2 элемента начиная с индекса 1 и добавляет 'kiwi' и 'grape' с индекса 1
console.log(fruits); // ['apple', 'kiwi', 'grape', 'strawberry']

map(): Этот метод создает новый массив, в котором каждый элемент изменен в соответствии с заданной функцией.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
let updatedFruits = fruits.map(fruit => fruit === 'banana' ? 'kiwi' : fruit);
console.log(updatedFruits); // ['apple', 'kiwi', 'orange']
console.log(fruits); // ['apple', 'banana', 'orange']

forEach(): Этот метод позволяет выполнить функцию для каждого элемента массива. Однако он не возвращает новый массив, поэтому для изменения элементов нужно использовать прямое присваивание.

let fruits = ['apple', 'banana', 'orange'];
fruits.forEach((fruit, index) => {
 if (fruit === 'banana') {
  fruits[index] = 'kiwi'; // Изменяет 'banana' на 'kiwi'
 }
});
console.log(fruits); // ['apple', 'kiwi', 'orange']

fill(): Этот метод заполняет все элементы массива статическим значением от начального до конечного индекса.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits); // ['apple', 'banana', 'orange']
fruits.fill('kiwi', 1, 2); // Заполняет элемент с индекса 1 до 2 значением 'kiwi'
console.log(fruits); // ['apple', 'kiwi', 'orange']

поиск элементов массива
indexOf(): Возвращает первый индекс, по которому данный элемент может быть найден в массиве, или -1, если элемент не найден.

let fruits = ['apple', 'banana', 'orange'];
let index = fruits.indexOf('banana'); // 1
console.log(index);

find(): Возвращает значение первого найденного элемента в массиве, который удовлетворяет предоставленной функции. Если ни один элемент не удовлетворяет условию, возвращает undefined.

let fruits = ['apple', 'banana', 'orange'];
let foundFruit = fruits.find(fruit => fruit.startsWith('b'));
console.log(foundFruit); // 'banana'

findIndex(): Возвращает индекс первого элемента в массиве, который удовлетворяет предоставленной функции. Если ни один элемент не удовлетворяет условию, возвращает -1.

let fruits = ['apple', 'banana', 'orange'];
let foundIndex = fruits.findIndex(fruit => fruit.startsWith('o'));
console.log(foundIndex); // 2

фильтрация элементов массива
filter(): Создает новый массив с элементами, которые прошли проверку, заданную в предоставленной функции.

let fruits = ['apple', 'banana', 'orange'];
let filteredFruits = fruits.filter(fruit => fruit.includes('n'));
console.log(filteredFruits); // ['banana', 'orange']

some(): Проверяет, удовлетворяет ли хотя бы один элемент массива условию, заданному в предоставленной функции. Возвращает true или false.

let fruits = ['apple', 'banana', 'orange'];
let hasBanana = fruits.some(fruit => fruit === 'banana');
console.log(hasBanana); // true

every(): Проверяет, удовлетворяют ли все элементы массива условию, заданному в предоставленной функции. Возвращает true или false.

let fruits = ['apple', 'banana', 'orange'];
let allHaveA = fruits.every(fruit => fruit.includes('n'));
console.log(allHaveA); // false

сортировка элементов массива
sort(): Этот метод сортирует элементы массива на месте и возвращает отсортированный массив. По умолчанию сортировка происходит по строковому значению.

let fruits = ['banana', 'apple', 'orange'];
fruits.sort(); // Сортирует массив по алфавиту
console.log(fruits); // ['apple', 'banana', 'orange']

Для сортировки чисел необходимо передать функцию сравнения:

let numbers = [10, 2, 5, 1, 8];
numbers.sort((a, b) => a - b); // Сортирует массив по возрастанию
console.log(numbers); // [1, 2, 5, 8, 10]

let numbers = [10, 2, 5, 1, 8];
numbers.sort((a, b) => b - a); // Сортирует массив по убыванию
console.log(numbers); // [1, 2, 5, 8, 10]

преобразование элементов массива
reduce(): Этот метод применяет функцию к аккумулятору и каждому элементу массива (слева направо), чтобы уменьшить его до одного значения.

let numbers = [1, 2, 3, 4];
let sum = numbers.reduce((accumulator, current) => accumulator + current, 0);
console.log(sum); // 10

flat(): Этот метод создает новый массив с элементами всех подмассивов, объединенными в один уровень.

let nestedArray = [1, [2, 3], [4, 5]];
let flatArray = nestedArray.flat();
console.log(flatArray); // [1, 2, 3, 4, 5]

map(): Как уже упоминалось, этот метод создает новый массив, в котором каждый элемент изменен в соответствии с заданной функцией.

let numbers = [1, 2, 3];
let doubled = numbers.map(num => num * 2);
console.log(doubled); // [2, 4, 6]

объединение массивов
concat(): Этот метод используется для объединения двух или более массивов.

let array1 = [1, 2];
let array2 = [3, 4];
let combined = array1.concat(array2);
console.log(combined); // [1, 2, 3, 4]

spread operator: Также можно использовать оператор распространения для объединения массивов.

let array1 = [1, 2];
let array2 = [3, 4];
let combined = [...array1, ...array2];
console.log(combined); // [1, 2, 3, 4]

итерация по элементам массива
forEach(): Этот метод выполняет указанную функцию один раз для каждого элемента массива.

let fruits = ['apple', 'banana', 'orange'];
fruits.forEach(fruit => {
 console.log(fruit);
});
// Вывод: apple, banana, orange

for...of: Этот цикл позволяет перебрать элементы массива, предоставляя доступ к каждому элементу.

let fruits = ['apple', 'banana', 'orange'];
for (let fruit of fruits) {
 console.log(fruit);
}
// Вывод: apple, banana, orange

for...in: Этот цикл используется для перебора индексов массива. Однако его лучше использовать для объектов, так как он может не работать должным образом с массивами.

let fruits = ['apple', 'banana', 'orange'];
for (let index in fruits) {
 console.log(fruits[index]);
}
// Вывод: apple, banana, orange

map(): Этот метод создает новый массив, содержащий результаты вызова указанной функции для каждого элемента массива.

let fruits = ['apple', 'banana', 'orange'];
let upperCaseFruits = fruits.map(fruit => fruit.toUpperCase());
console.log(upperCaseFruits); // ['APPLE', 'BANANA', 'ORANGE']

примененик итерации

Итерация по массивам позволяет выполнять различные операции, такие как:

- Применение функций к каждому элементу массива.

- Изменение значений элементов.

- Сбор данных в новый массив или объект.

Выбор метода итерации зависит от конкретной задачи и желаемого результата. Например, если нужно просто выполнить действие для каждого элемента, лучше использовать forEach(). Если нужно создать новый массив на основе существующего, то подойдет map().

методы проверки массивов
includes(): Этот метод проверяет, содержит ли массив определённый элемент, и возвращает true или false.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits.includes('banana')); // true
console.log(fruits.includes('grape')); // false

indexOf(): Этот метод возвращает первый индекс, по которому данный элемент может быть найден в массиве, или -1, если элемент не найден.

let fruits = ['apple', 'banana', 'orange'];
console.log(fruits.indexOf('banana')); // 1
console.log(fruits.indexOf('grape')); // -1

find(): Этот метод возвращает значение первого элемента массива, который удовлетворяет предоставленной функции. Если ни один элемент не удовлетворяет, возвращает undefined.

let numbers = [1, 2, 3, 4, 5];
let found = numbers.find(num => num > 3);
console.log(found); // 4

some(): Этот метод проверяет, удовлетворяет ли хотя бы один элемент массива условию, заданному в предоставленной функции. Возвращает true или false.

let numbers = [1, 2, 3, 4, 5];
let hasEven = numbers.some(num => num % 2 === 0);
console.log(hasEven); // true

every(): Этот метод проверяет, удовлетворяют ли все элементы массива условию, заданному в предоставленной функции. Возвращает true или false.

let numbers = [2, 4, 6, 8];
let allEven = numbers.every(num => num % 2 === 0);
console.log(allEven); // true

применение

Методы проверки массивов позволяют:

- Определять наличие элементов в массиве.

- Находить элементы по заданным критериям.

- Проверять условия для одного или всех элементов массива.

Выбор метода проверки зависит от конкретной задачи. Например, если нужно узнать, есть ли элемент в массиве, лучше использовать includes(). Если нужно найти элемент по условию, подойдет find().

копирование массивов
Метод slice(): Этот метод возвращает новый массив, содержащий копию части исходного массива. Если не передать аргументы, он создаст полную копию массива.

let originalArray = [1, 2, 3, 4, 5];
let copiedArray = originalArray.slice();
console.log(copiedArray); // [1, 2, 3, 4, 5]

Оператор распространения (spread operator): С помощью оператора ... можно создать новый массив, который будет содержать элементы другого массива.

let originalArray = [1, 2, 3, 4, 5];
let copiedArray = [...originalArray];
console.log(copiedArray); // [1, 2, 3, 4, 5]

метод преобразования массивов в строки
join(): Этот метод объединяет все элементы массива в строку, используя указанный разделитель. Если разделитель не указан, по умолчанию используется запятая.

let fruits = ['apple', 'banana', 'orange'];
let result = fruits.join(', ');
console.log(result); // "apple, banana, orange"

toString(): Этот метод возвращает строковое представление массива, элементы которого также разделены запятыми.

let numbers = [1, 2, 3, 4, 5];
let result = numbers.toString();
console.log(result); // "1,2,3,4,5"

String(): Преобразует массив в строку, используя метод toString().

let mixedArray = [1, 'apple', true];
let result = String(mixedArray);
console.log(result); // "1,apple,true"

применение

Преобразование массивов в строки позволяет:

- Формировать удобочитаемые представления данных для отображения.

- Подготавливать данные для передачи в другие функции или API.

- Упрощать работу с текстовыми данными, когда необходимо объединить элементы массива в одну строку.

Выбор метода преобразования зависит от конкретной задачи. Например, если нужно создать строку с определённым разделителем, лучше использовать join(). Если же требуется просто получить строковое представление массива, можно воспользоваться toString().

методы преобразования строк в массив
Метод split(): Этот метод разбивает строку на массив подстрок, используя указанный разделитель. Если разделитель не указан, строка будет разбита на массив, содержащий один элемент — саму строку.

let str = "apple,banana,orange";
let fruitsArray = str.split(",");
console.log(fruitsArray); // ["apple", "banana", "orange"]

Метод Array.from(): Этот метод создает новый массив из массивоподобного или итерируемого объекта. Например, можно использовать его для преобразования строки в массив символов.

let str = "hello";
let charArray = Array.from(str);
console.log(charArray); // ["h", "e", "l", "l", "o"]

Оператор распространения (spread operator): С помощью оператора ... можно преобразовать строку в массив символов.

let str = "world";
let charArray = [...str];
console.log(charArray); // ["w", "o", "r", "l", "d"]

применение

Преобразование строк в массивы позволяет:

- Разделять текстовые данные на отдельные элементы для дальнейшей обработки.

- Упрощать манипуляции с текстом, например, при анализе или изменении отдельных символов или слов.

- Подготавливать данные для работы с массивами, когда необходимо выполнять операции, такие как фильтрация или сортировка.

Выбор метода преобразования зависит от конкретной задачи. Например, если нужно разбить строку на слова или элементы, лучше использовать split(). Если требуется получить массив символов, можно воспользоваться Array.from() или оператором распространения.

ОБЪЕКТЫ
Определение
  • В JavaScript объекты представляют собой коллекции свойств, где каждое свойство состоит из ключа и значения. Объекты могут содержать различные типы данных, включая строки, числа, массивы и даже другие объекты. Они являются основным способом организации и хранения данных в JavaScript.
Создание объектов
литерал объекта
  • let person = {
     name: 'Иван',
     age: 30,
     isStudent: false
    };
  • console.log(person); // { name: 'Иван', age: 30, isStudent: false }
конструктор объекта
  • let person = new Object();
    person.name = 'Иван';
    person.age = 30;
    person.isStudent = false;
  • console.log(person); // { name: 'Иван', age: 30, isStudent: false }
классы (ES6)
  • class Person {
     constructor(name, age) {
      this.name = name;
      this.age = age;
     }
    }
  • let person = new Person('Иван', 30);
    console.log(person); // { name: 'Иван', age: 30, isStudent: false }
Доступ к свойствам
свойства объекта можно получить с помощью точечной нотации или квадратных скобок:
  • let person = {
     name: 'Иван',
     age: 30
    };
  • console.log(person.name); // 'Иван'
  • console.log(person['age']); // 30
Изменение свойств
свойства объекта можно изменять с помощью точечной нотации или квадратных скобок:
  • let person = {
     name: 'Иван',
     age: 30
    };
  • person.age = 31;
  • console.log(person); // {name: 'Иван', age: 31}
  • person['age'] = 32;
  • console.log(person); // {name: 'Иван', age: 32}
Добавление и удаление свойств
свойства объекта можно добавлять и удалять с помощью точечной нотации или квадратных скобок:
  • let person = {
     name: 'Иван',
     age: 30
    };
  • person.gender = 'мужской'; // person['gender'] = 'мужской'; добавление свойства
  • console.log(person); // {name: 'Иван', age: 30, gender: 'мужской'}
  • delete person.age; // delete person['age']; удаление свойства
  • console.log(person); // {name: 'Иван', gender: 'мужской'}
